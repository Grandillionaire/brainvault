# contributing to brainvault

thanks for your interest in contributing! here's how you can help.

## getting started

1. fork the repo
2. clone your fork: `git clone https://github.com/YOUR_USERNAME/brainvault`
3. create a branch: `git checkout -b fix-something`
4. make your changes
5. test locally: `npm run dev`
6. commit: `git commit -m "fix: description of fix"`
7. push: `git push origin fix-something`
8. open a pull request

## development setup

```bash
npm install
npm run dev
```

the app runs at http://localhost:5173

## code style

- use existing patterns from the codebase
- run tests before submitting: `npm test`
- keep it simple and fast

## commit messages

use lowercase, descriptive messages:
- `fix: editor not saving on blur`
- `add: export to pdf feature`
- `update: improve search performance`

## what to contribute

- bug fixes
- new features (discuss in an issue first)
- documentation improvements
- performance optimizations
- tests

## what not to do

- add unnecessary dependencies
- break existing functionality
- ignore test failures
- add tracking or analytics

## questions?

open an issue and we'll help you out.
